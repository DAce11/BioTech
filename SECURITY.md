# Security Policy for Bionic Arm Web Controller

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly:

1. **Do not create a public issue** on GitHub.  
2. Contact the project maintainers directly at: **cincorinold@gmail.com**  
   (replace this with your actual email address)  
3. Include as much detail as possible, such as:
   - Steps to reproduce the issue
   - Environment details (OS, browser, NodeMCU version, etc.)
   - Screenshots or logs if applicable
4. We will acknowledge your report within **48 hours** and provide updates as we work on a fix.

---

## Supported Versions

- Only the latest release and the previous release are officially supported for security updates.  
- Older releases may not receive security fixes.

---

## Security Practices

- Never share sensitive credentials publicly (Firebase API keys, WiFi passwords, or user passwords).  
- Users should login only with pre-created accounts; no self-registration.  
- All communication with the NodeMCU device should be on a secured network whenever possible.

---

## Additional Notes

- By reporting a security issue, you help protect other users of this project.  
- If you’re unsure whether something is a vulnerability, it’s better to **contact the maintainers first**.
